# 章节结构与场景组织指南

[概述]
    本文件指导单章的内部结构设计。一章不是一段连续长文，而是由若干场景单元组成的叙事段落，每个场景有独立的功能和节奏定位。

    章节字数范围：3000-5000字，最低不低于2500字
    场景数量：根据内容决定，不设固定上限
    核心事件数量：1-2个主要事件

[场景单元设计]
    每个场景应包含三个要素：
    - 目标：这个场景要推进什么（信息/情感/关系/冲突）
    - 转变：进入场景和离开场景时，情况发生了什么变化
    - 衔接：如何自然过渡到下一个场景

    如果一个场景没有明确的"转变"，考虑是否真的需要它。
    如果两个场景的功能高度重叠，考虑合并为一个场景。

    场景之间的关系类型：
    - 因果型：场景A的结果触发场景B
    - 对比型：场景A的氛围与场景B形成反差
    - 并行型：两条线索同时推进，交叉呈现
    - 递进型：同一冲突在不同场景中逐步升级

    场景过渡方式：
    - 时间跳跃："三天后" — 简洁直接，适合节奏紧凑段落
    - 空间切换：跟随角色移动自然转场
    - 情绪桥接：上一场景的情绪余韵延伸到下一场景开头
    - 硬切：直接切换，不做过渡，制造冲击感

[开篇切入]
    章节的前20%内容对读者留存影响最大。优先在开篇建立紧张感、呈现重大事件或制造情感冲击。

    以下开头方式容易让读者失去兴趣：
    - 天气描写起手（"那天天气晴朗..."）
    - 日常流程铺陈（"他醒来，刷牙，吃早餐..."）
    - 回顾上章内容（"上一章说到..."）
    - 大段背景介绍（"先介绍一下这个城市..."）
    - 无信息量的寒暄对话

    更有效的切入方式：

    [动作切入]
        角色正处于某个行动中，读者被直接拉入现场。
        示例：子弹擦过他的耳边，击碎了身后的花瓶。李明没有回头，翻滚躲到沙发后面。

    [对话切入]
        一句有信息量或冲击力的对话直接建立情境。
        示例："我把孩子卖了。" 妻子平静地说，继续翻阅杂志。

    [反常情境]
        呈现一个不符合常理的场景，激发好奇。
        示例：死人坐在办公桌前，正在写一份报告。

    [悬念切入]
        从结果倒推，先展示异常再解释原因。
        示例：三天后，所有人都会后悔今天的决定。但此刻，会议室里的每个人都在微笑。

    [危机切入]
        从角色面临重大威胁的时刻开始。
        示例：门被踹开的瞬间，她知道藏不住了。保险箱里的秘密即将曝光。

    [发现切入]
        从发现关键线索或真相开始。
        示例：法医报告摊在桌上。死因不是意外，是精心策划的谋杀。而嫌疑人只有一个——他自己。

    开篇的核心任务：
    - 让读者快速知道"在哪、是谁、正在发生什么"
    - 同时制造一个疑问或张力，驱动继续阅读
    - 如果写完开头感觉平淡，尝试砍掉前两段，从第三段开始

[章节主体结构]
    标准章节由四个功能区间组成，比例可根据需要调整：

    [开头钩子] 约占前20%
        建立冲突、呈现事件、制造情感冲击。
        使用上述切入方式之一或组合使用。

    [发展推进] 约占中间50-60%
        本章的核心内容区。每个场景应至少完成以下一项：
        - 揭示新信息：让读者或角色获得重要信息
        - 改变关系：人物关系发生转变（结盟、背叛、亲密、疏远）
        - 升级问题：现状恶化，新障碍出现
        - 推动角色：角色做出选择、获得认知、承受后果

        如果某段内容删掉后不影响理解，考虑是否需要保留。

    [高潮时刻] 约占后15-20%
        本章的情感或事件最高点：
        - 动作高潮：战斗、追逐、对抗的决定性瞬间
        - 情感高潮：重大发现、背叛、告白、牺牲
        - 认知高潮：角色或读者对局势的理解发生根本转变

    [结尾钩子] 约占最后5-10%
        留下悬念，制造翻页欲望。
        详细技巧参考 shared/hook-techniques.md

        常见收尾类型：
        - 悬念型：揭露新信息或新威胁，戛然而止
        - 反转型：最后一句颠覆本章建立的认知
        - 情感型：角色做出重大决定或情感爆发
        - 平静型：表面平静但暗含不安（暴风雨前的宁静）

        避免每章都用同一种收尾模式。
        避免把所有事情交代完毕让读者没有理由翻页。

[章节类型]
    不同章节承担不同叙事功能，明确类型有助于控制节奏：

    [情节推进章]
        目的：推动主线剧情发展
        特征：有明确的事件进展、重要信息揭示
        如：主角发现线索、敌人发动攻击、盟友背叛

    [人物深化章]
        目的：深化读者对人物的理解
        特征：揭示人物背景、动机、内心冲突
        如：回忆片段、私密对话、独处时刻
        注意：应与主线相关，避免变成孤立的人物小传

    [氛围营造章]
        目的：建立特定情绪或紧张感
        特征：注重感官描写、节奏控制
        如：暴风雨前的宁静、潜伏行动、等待审判

    [过渡衔接章]
        目的：连接两个重大事件
        特征：信息整理、位置转换、时间跳跃
        注意：保持简洁，过渡章容易拖沓

    一章可以兼具多种类型，但应有一个主要功能。

[段落编排]
    段落长度服务于节奏：
    - 长段落（150-300字）：沉浸、内心活动、环境铺陈
    - 中段落（50-150字）：叙事推进、信息交代
    - 短段落（1-50字）：冲击、转折、情绪爆发

    节奏原则：
    - 避免连续多个相同长度的段落
    - 高压场景用短段落加速
    - 情感场景用长短交替制造呼吸感
    - 转折点前适当放慢，转折时突然加速

    信息密度交替：
    - 高密度（动作、对话密集、事件快速推进）和
      低密度（内心活动、环境描写、情感沉淀）交替出现
    - 持续高密度导致读者疲劳
    - 持续低密度导致读者无聊

[视角管理]
    单章内的视角处理：
    - 优先保持单一视角，减少跳转
    - 如需切换视角，以场景为单位切换，避免在场景中间切
    - 视角人物优先选择本场景中"利益最大"的角色
    - 非视角人物的内心通过行为、对话、微表情间接呈现

[章节自查]
    写完拼合后对照输出，不得跳过：

    「章节自查」

    开篇：
    - 切入方式：[使用了哪种切入方式]
    - 前20%张力：[有效 / 平淡，说明原因]
    - 如平淡：[是否尝试从第三段重新切入]

    内容：
    - 本章主要功能：[情节推进 / 人物深化 / 氛围营造 / 过渡]
    - 每个场景的转变：
      - 场景1：[A→B，是否发生]
      - 场景2：[A→B，是否发生]
    - 可删段落：[有 / 无，如有列出位置]

    结尾：
    - 收尾类型：[悬念 / 反转 / 情感 / 平静]
    - 钩子是否到位：[是 / 否，说明]
    - 与前几章收尾的差异：[说明]

[骨架与正文的边界]
    骨架稿和正文有明确的区分标准。违反此边界是常见的结构错误。

    骨架稿应该像什么：
    - "张远推门进来。'人找到了。' 李默放下茶杯。'在哪？' '地下室。'"
    - 只有动作和关键台词，读者能理解情节推进

    骨架稿不应该像什么：
    - "张远神色紧张地推门进来，额头上还挂着汗珠..."（感官描写混入）
    - "这里展现了两人的信任危机"（规划语言）
    - "本场景旨在揭示李默的谎言"（目标说明而非叙事）

    正文中禁止的骨架残留模式：
    - "本章将展示..." / "这个场景揭示了..."（规划元语言）
    - "接下来我们要看到..."（跳出叙事）
    - "通过这个事件，读者将理解..."（教学口吻）
    - 列表式结构（第一、第二、第三）出现在叙事中

    扩展阶段必须做到：
    - 骨架中的"谁做了什么"变成具体的叙事语言
    - 骨架中的关键台词扩展为完整对话场景
    - 所有规划语言必须转换为叙事表达

[情感弧执行]
    章节规划中定义了读者情感弧（入口X → 出口Y），写作时必须有意识执行。

    执行方法：
    - 扩展场景时，关注每个场景的情感转变是否与整体情感弧一致
    - 场景的情感转变是情感弧的一个片段，不是独立的情绪岛
    - 重章（紧张-释放/渐进-爆发）：逐场景累积张力，高潮处释放
    - 中章（困惑-清晰）：信息逐步释放，读者与角色同步接近真相
    - 轻章（舒适-颠覆-新平衡/情感着陆）：先建立安全感再打破，或收束情绪

    情感弧检查（拼合阶段必做）：
    - 入口状态是否与规划定义的读者情感入口一致
    - 出口状态是否与规划定义的读者情感出口一致
    - 如果偏差超过一个情绪层级（如平静→恐慌 vs 平静→不安），需要修正

    情感弧不是限制创意，而是确保章节有清晰的情感旅程。
    如果写作中发现了更好的情感走向，记录原因，reviewer 会判断是否有效。

[体裁感知]
    写作前读取 outline.md 中的体裁字段，参考 references/genre-standards.md 中对应体裁的专用标准。
    不同体裁对以下方面的优先级不同：
    - 信息控制 vs 情感深度
    - 动作描写 vs 心理描写
    - 对话密度 vs 环境铺陈
    - 节奏偏好

    体裁标准不改变骨架/正文边界，不改变三阶段写作流程。
    体裁影响的是扩展阶段的侧重点和拼合检查的优先级。

[项目积累]
    来自《大荒图》ch01 人类反馈（2026-03-10）：

    1. 开头切入规范
       - 开头前3段必须切入异常/冲突，禁止从"走路感受环境"开始
       - 环境应在行动中顺带呈现
       来源：进入故事太慢

    2. 悬念设置规范
       - 悬念应具体紧迫，与主角目标相关
       - "为什么这么冷清"太泛；"三尾长老为何只剩三尾"更具体
       - 读者需要"会发生什么"而非"这是什么"
       来源：缺乏悬念钩子

    3. 信息密度规范
       - 删除纯寒暄对话（喝茶、客套）
       - 每段对话或场景必须推动剧情（揭示信息、改变关系）或揭示人物
       来源：信息密度低

    4. 情感 reveal 规范
       - 重大情感转折（如认出亲人）前需有前置伏笔
       - 不能突然"轰然一声"回忆杀
       - 读者应和角色一起"快要想起但想不起"
       来源：情感转折突兀

    5. 主角主动性规范
       - 主角必须有内在动机驱动行动，而非被动等待事件
       - 动机可以是目标（寻找真相）或情感（关心某人）
       来源：主角零主动性
