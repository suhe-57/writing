# 体裁质量标准

> 不同体裁有不同的优先级和常见陷阱。
> Writer 在扩展阶段参考对应体裁的专用标准。
> Reviewer 在审查时根据体裁调整维度权重。

---

## 武侠

**核心优先级：**
1. 武打场面的动作清晰度（谁对谁、用什么招式、结果如何）
2. 侠义主题的内在一致性（角色的行为符合其侠义观）
3. 古典节奏感（句式偏短，对仗感，留白多于直白）
4. 暴力节制（点到为止，不过度渲染血腥）

**对话风格：**
- 偏文白夹杂，但不生硬
- 江湖切口适度（切口太多影响可读性）
- 不同门派有说话风格差异

**描写侧重：**
- 动作描写：招式拆解、身法、兵器
- 环境：山水意境、客栈、庭院
- 情感：克制表达，重行动轻告白

**常见陷阱：**
- 武打变成招式列表（"他用了A招，然后B招，然后C招"）→ 应写战斗节奏和感官
- 侠义说教化（角色大段阐述正义）→ 侠义应通过选择体现，不是台词
- 现代语感混入（"她的内心很崩溃"）→ 用符合时代感的表达

**节奏偏好：**
- 武打场景：重，短段落加速
- 情感场景：轻，长段落铺陈
- 典型模式：重-轻-重-轻（波浪式）

## 仙侠

**核心优先级：**
1. 修炼体系的内在一致性（境界、灵力、功法不自相矛盾）
2. 尺度感（力量层级清晰，升级有代价）
3. 意象的仙侠感（灵气、天劫、道心、因果）
4. 超越主题（从凡人到超脱的弧光）

**对话风格：**
- 可偏古风和玄幻之间的平衡
- 高阶修士说话有分量感（不啰嗦）
- 道/法相关讨论有哲理但不空洞

**描写侧重：**
- 战斗：能量流动、法则碰撞、天地异象
- 修炼：内视、感悟、瓶颈突破
- 环境：仙山、洞府、秘境、天劫

**常见陷阱：**
- 战力膨胀（角色无代价地突破）→ 每次突破需铺垫和代价
- 设定堆砌（大段解释修炼体系）→ 设定通过冲突自然揭示
- "天道"等概念模糊化→ 需有具体规则，不是万金油

**节奏偏好：**
- 突破/天劫：重
- 游历/寻宝：中
- 论道/静修：轻
- 典型模式：轻-中-重-中-轻（三明治式）

## 悬疑

**核心优先级：**
1. 信息控制（读者知道什么、角色知道什么、何时揭示）
2. 误导的公平性（读者有足够线索推出真相，但被合理误导）
3. 拼图满足感（揭秘时所有碎片能严丝合缝拼上）
4. 节奏紧迫感（悬念不断刷新，读者不停翻页）

**对话风格：**
- 信息密度高，每句对话都有功能
- 角色不说多余的话（除非是为了故意误导）
- 审讯/对质场景：问答节奏快，打断、试探、回避

**描写侧重：**
- 异常细节（不协调的物品、表情、时间差）
- 光影对比（明暗交替营造不安感）
- 不和谐音（日常中的异常点）

**常见陷阱：**
- 信息倾销（一章塞太多线索）→ 线索分散，每章1-2个核心线索
- 不公平误导（真相全靠最后一章的新信息）→ 所有关键线索提前埋设
- 悬念疲劳（每章都用"突然..."）→ 悬念类型交替，参考 hook-techniques.md

**节奏偏好：**
- 解谜章：重
- 线索收集章：中
- 过渡/整理章：轻
- 典型模式：中-重-轻-重-中（波浪式）
- 接近高潮时：轻-中-中-重（爬坡式）

## 言情

**核心优先级：**
1. 情感真实性（角色的情感转变有内在逻辑，不是突然爱上）
2. 对话中的潜台词（角色不说出真实感受，通过言行缝隙传达）
3. 关系进展的可信度（从陌生到亲密有渐进阶段）
4. 克制中的张力（不说破时的暗流涌动）

**对话风格：**
- 话中有话，表面和内心不同
- 情绪标签清晰（赌气、试探、吃醋、逞强）
- 亲密关系变化通过说话方式变化体现

**描写侧重：**
- 微表情和肢体语言
- 空间距离（靠近、回避、无意触碰）
- 感官细节（气味、温度、触觉比视觉更有代入感）

**常见陷阱：**
- 情感跳跃（"她突然意识到自己爱上了他"）→ 需要渐进式累积
- 过度直白（角色直接说"我喜欢你"）→ 早期用潜台词，后期才直白
- 配角工具化（配角只为推动男女主感情存在）→ 配角有自己的动机

**节奏偏好：**
- 关系转折章：重
- 日常互动章：轻
- 冲突章：中
- 典型模式：轻-轻-中-重-轻（爬坡+释放）

---

## 科幻

**核心优先级：**
1. 概念的清晰表达（不灌水地解释技术设定）
2. 推测一致性（设定的科技规则不自相矛盾）
3. 人性 stakes（技术背景下的个人命运和情感）
4. 世界观沉浸感（未来感/异质感的具体呈现）

**对话风格：**
- 技术术语自然融入，不大段解释
- 不同角色对技术的态度差异（乐观/悲观/实用）
- 专业角色的语言有精确性

**描写侧重：**
- 技术环境的感官化描写（不是参数列表）
- 社会结构的具象化（通过具体场景展现，不是社会学论文）
- 人与技术的互动界面

**常见陷阱：**
- Info-dump（连续段落解释设定）→ 设定通过冲突/问题自然揭示
- 技术压倒人性（角色变成技术讨论的工具人）→ 保持人性 stakes 在前
- "未来感"靠堆砌名词→ 用具体的日常细节建立异质感

**节奏偏好：**
- 概念揭示章：中-重
- 人物章：轻-中
- 行动/危机章：重
- 典型模式：轻-中-重-中-轻（三明治式）
