# 流程验证与迭代记录

本 Skill 建议用以下场景做流程验证；验证时记录迭代点，便于后续微调。

---

## 场景 1：开题调研

- **用例**：用户说「想写言情，男主旅途邂逅女主，先帮我做大纲调研」。
- **验证点**：
  - 是否按阶段 1→6 顺序追问，有无跳阶段或过早出稿。
  - 每阶段是否用 checklists 做缺口扫描，缺口时是否定向补问。
  - 最终输出是否包含：创作目标、双主角关系主线、分幕大纲、时间线草图、人物草图、风险与待确认项。
- **迭代记录**：若追问过少/过多或输出缺项，调整 checklists 或 output-templates。

---

## 场景 2：续写 / 改写 / 章节规划快线

- **用例**：用户说「改第 X 章到第 Y 章，先帮我理一遍」或「帮我理一下后面几章怎么走」。
- **验证点**：
  - 是否**先读** meta、outline、characters、timeline 及目标章前后文（含已有正文）。
  - 是否**先复述**（3–5 句）已读内容与理解到的意图。
  - 是否**先追问** 1–3 个关键问题，等用户确认后再给方案。
  - 改写时是否先问：保留项、不满项、小修/中调/重构。
  - 默认输出是否为**短版**：每章标题 + 200–300 字内容介绍；用户说「展开」才出长报告。
- **迭代记录**：若跳过阅读或追问、或输出过长，调整 SKILL §2 与 output-templates。

---

## 场景 3：类型泛化（古代/悬疑/科幻等）

- **用例**：古代、悬疑、科幻等非「现代言情」类型。
- **验证点**：阶段 1–6 是否仍适用；checklists 中的「双主角」「俘获芳心」等术语是否已按类型替换为对应表述；output-templates 中的开题报告结构是否适配该类型。
- **迭代记录**：若某类型下缺关键维度，在开题清单中增加可选维度；若输出模板需类型专属字段，可增补变体。

---

验证完成后，将迭代点反映到 SKILL.md、checklists.md、output-templates.md，并保留本文件作为对照。
