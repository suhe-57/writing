# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 交流语言

始终使用中文与用户交流。

## 项目性质

这是一个 Claude Code 写作技能仓库，不是代码项目。所有内容位于 `.claude/skills/` 下，均为 Claude Code Skill 定义文件（Markdown 格式）。没有编译、构建、测试或 lint 命令。

## 核心架构：两种写作路径

```
用户写作意图
    ├── 短篇 / 观点类（1000-15000 字）
    │   ├── 批判性文章 ──→ ljg-writes（观点层层剥开，1000-1500 字）
    │   ├── 通用文章 ────→ writing-assistant（诊断→选题→框架→内容）
    │   └── 短篇网文 ────→ story-short-write（情绪驱动，8000-20000 字）
    │
    └── 长篇 / 小说类
        ├── 开题 / 理大纲 ──→ novel-outline-researcher（先读、先问、再给方案）
        ├── 架构设计 ──────→ novel-worldbuilding（大纲、人物、世界观、伏笔）
        └── 逐章执笔
            ├── novel-prose（沉浸式创作，单章 ≥10000 字符）
            └── writer-skill（三段式：骨架→扩展→拼合，单章 ≥2500 字）
```

## 各 Skill 职责与边界

### ljg-writes（批判性文章写作引擎）

- **定位**：像手术刀剖开一个观点，层层剥到底。边想边写，不是要点罗列。
- **产出**：1000–1500 字，Org-mode 格式，输出到 `~/Documents/notes/{时间戳}==z--{标题}__write.org`。
- **铁律**：不自标深度（禁用"再深入一层"等宣告）；翻译腔免疫；同一个句式全文最多出现一次；写完逐段朗读检验口语感。

### writing-assistant（通用写作助手）

- **定位**：帮用户从模糊想法走到成文，分 5 个阶段执行。
- **流程**：阶段00（诊断观点是否清晰）→ 清晰走框架→内容；模糊走挖掘→选题→框架→内容。
- **每个阶段开始必须**：告诉用户当前在哪个阶段、目标是什么、会做什么事。
- **模块化**：框架讨论（03）和内容产出（04）是两个通用模块，两条分支共享。

### story-short-write（短篇网文）

- **定位**：情绪驱动的短篇小说，从构思到成稿，四个阶段。
- **核心铁律**：短篇写的是情绪，不是故事；一个反转撑一篇；默认第一人称。
- **格式规范**（最高优先级）：一句一段、不超 60 字、段间无空行、无缩进、对话独立成行引号 `""`、数字编号分割小节。
- **字数纪律**：全文 ≥8000 字，每节 ≥800 字，节数必须等于大纲规划节数。
- **输出目录**：`{标题}/` 下，含 `设定.md`、`小节大纲.md`、`正文.md`。

### novel-outline-researcher（小说大纲调研）

- **定位**：调研与规划，不写章节正文。支持言情/古代/悬疑/科幻等类型。
- **核心流程**：先读（Step 0 强制读取前文和设定）→ 复述确认（Step 1）→ 追问 1-3 个关键问题（Step 2）→ 等用户确认（Step 3）→ 输出章节方案（Step 4）。
- **输出模式**：
  - 开题调研：长报告，6 阶段追问直到 95% 确定。
  - 续写/改写/章节规划：**默认短版**（每章标题 + 200–300 字简介），用户说"展开"才出长报告。
- **改写必须先问 3 件事**：保留什么、不满意什么、小修/中调/重构。
- **报告落盘**：`reports/` 目录（仅当用户明确要求保存）。

### novel-worldbuilding（小说架构设计）

- **定位**：设计小说整体架构，专注结构模板和设计方法论。
- **工作流**：需求分析 → 世界观构建 → 角色设计 → 情节架构 → 输出 plan.md。
- **输出目录**：`novel/{书名}/`，含 `plan.md`、`正文/`、`summary/`，设定放 `设定/`。

### novel-prose（小说写作技法）

- **定位**：逐章执笔，沉浸式创作。单章 ≥ 10000 中文字符。
- **写作流程**：场景规划（5-8 个场景）→ 逐场景写作 → 整章整合。
- **文件输出**：`novel/{书名}/正文/第X章_{标题}.md`。
- **核心约束**：视角一致、对话符合角色、不假设设定（必须查询设定文件）。

### writer-skill（写作技能包）

- **定位**：章节正文写作的三段式方法（骨架→扩展→拼合）。
- **阶段一（骨架稿）**：只写核心动作和关键台词，300-400 字/场景。强制输出"骨架自检"。
- **阶段二（分场景扩展）**：在骨架基础上逐场景扩展，添加感官、心理、环境。单场景 ≤1500 字。强制输出"场景完成确认"。
- **阶段三（拼合检查）**：所有场景拼合。总字数 ≥2500 字。强制输出"拼合检查"。
- **输出**：`plans/chNN-skeleton.md`（骨架稿），`chapters/chNN.md`（正文）。
- **与 novel-prose 的区别**：writer-skill 强调结构化流程和自检清单；novel-prose 强调沉浸式创作心态。

## 文件命名与输出约定

| Skill | 输出路径 | 命名格式 |
|-------|---------|---------|
| ljg-writes | `~/Documents/notes/` | `{时间戳}==z--{标题}__write.org` |
| story-short-write | `{标题}/` | `设定.md`, `小节大纲.md`, `正文.md` |
| novel-worldbuilding | `novel/{书名}/` | `plan.md` |
| novel-prose | `novel/{书名}/正文/` | `第X章_{标题}.md` |
| writer-skill | `chapters/`, `plans/` | `chNN.md`, `chNN-skeleton.md` |
| novel-outline-researcher | `reports/` | `{书名}-{类型}报告-{YYYYMMDD}.md` |
